Statistics

Created Friday 22 January 2016

Types of statistical analysis

lama currently provides two types of statistical analysis.

  • Linear model
  • Simple ttest

The statistics work on several of the data outputs

  • Intensity (optional normalised to the background)
  • deformation fields
  • jacobians

pheno_detect.py (see here) creates a folder called 'stats' and a config file within it called stats.yaml.

This file is used by lama_stats.py in the statistical analysis . lama_stats.py is run automatically by pheno_detect.py, but can also be run on its own.

$ lama_stats.py -c stats.yaml


Config file



For each data output type we specify:

  • The path to the respective wild type and mutant data output folders
  • The statsitical test to perform. Options are:
    • lmR linear model (Required R to be installed
    • ttest (does not need R, but is currently broken and will be fixed soon)
  • The fixed mask that was specified in the original wild type test test config file.
    • This file is used to mask regions of the images we are not interested in, which speeds up analysis and helps prevents over-correcting the p-values in the FDR step
  • formulas: A list of formulas. The default 'data ~ genotype' might be all you need, but more can be added such as 'data ~ genotype + sex'. If more ? are added than this needs to be specified in the groups.csv file.
  • voxel_size: The physical size of the voxels in um. This is used to calculate voxel sizes of the inverted labels (not currently used and total volume is given in voxels)
  • n1: Wheter to perform n1(mutant sample size of 1) analysis
    • default = True

Stats outputs


The stats folder is created in the output director and minimally contains the following


Look inside the jacobian folder, we have two statistical output files:

  • T-statistics overlays end in FDR_0.05_stats_.nrrd
    • At each pixel we have a t-statistic and corresponding p-value.
    • The p-values are corrected for multiple testing to get q-values
    • Any position where the q-value is < 0.05, the corresponding t-statistic is output to the volume, otherwise the value is set to 0
  • T-statistic/q-value overlays
    • These filenames end with t_q_stats_.npz
    • The q-values are derived as above.
    • No filtering occurs, and the q-values, and t-statitistics are output into a joint volume of npz format
    • This .npz file can be viewed in inb vpv (link to public github page) where filtering on q-value can be done onthe fly.

Specimen sample size of 1 (n1)

Variably penetrent phenotypes may not be identified with the statistical anlaysis described above. This is especially so with a low mutant sample number.
Because of this, a seperate analysis is performed:

  • At each voxel position the range of wild type values are determined. A z-score (number of standard deviations from the wild type mean) is calculated
  • If the z-score > 3, the zscore is output output the result volume. Otherwise the result volume voxel is set to 0
  • There will be a an output volume from each mutant specimen

Inversion of stats output

The statistical outputs can be overlaid onto the population average target or the registrered images.
However, it can be useful to see the results overlaid onto the original images. By default, the results are inverted back to the rigidly-aligned images.
