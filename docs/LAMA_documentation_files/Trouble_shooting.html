Trouble shooting

Created Monday 01 February 2016

Error messages

Some error messages that you may see in the log file


16 bit voxel type error

error:
If using 16 bit input volumes, 'FixedInternalImagePixelType' and 'MovingInternalImagePixelType should be set to 'float' in the global_elastix_params secion of the config file

explaination
The internal representation of the voxels that elastix uses during registration can be varied. We tend to use short on 8bit input volumes as this speeds up processing time.
However, we found this to produce artifacts on the output images when used in conjunction with 16bit images. So when using 16bit input images, set FixedInternalImagePixelType' and 'MovingInternalImagePixelType to float in the main config file


Image padding error

error:
can't pad image. {image name} is larger than the specified volume size.

explaination:
All the input images, label maps, masks etc. must all be the same dimensions. To make sure they are, all images are padded to either the largest dimensions within all the input images, or it is specified with the pad_dims entry in the config file.
When the dimensions are specified, if an input image is larger in any dimension than that specified, we get this error.
This can happen if a phenodetect.py run is being performed in which a mutant is larger in one or more dimension than one if the wild type test set.
To remedy this, the wild type test set should be run again with lama.py but with pad_dims set to be larger or equal to any of the mutant image dimensions
