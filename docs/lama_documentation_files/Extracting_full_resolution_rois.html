Extracting full resolution rois

Created Tuesday 26 July 2016

Registration is normally carried out on scaled volumes in order to save computer resources and to speed up the process.
Once registration has completed and all our volumes are in the same space, it is sometimes desirable to obtain regions of interest from our orginal unscaled images.
These full resolution rois can be used for viaual analysis or can be put through LAMA to automatically detect abnormal phenotypes at native resolution

There are several step to do this


Add info about the full resoution images to the LAMA config

Enter the paths to your full resolution images in the top section of main config file before running lama 


# Info for extracting rois from the original data
full_resolution_folder: /home/neil/mousedata/MicroCT/project-IMPC_MCT/processed_recons
full_resolution_subfolder_name: cropped
log_file_endswith: rec.log
voxel_size_log_entry: 'Pixel Size (um)='


full_resolution_folder

This is an absolute path a folder containing your full resolution images (within subfolder)

full_resolution_subfolder_name

If you have another folder within your image folder, for example a cropped folder, it can be specified here

log_file_endswith

The voxel size of the full size images can be extracted from a log file for each volume. This entry specifies a pattern that identifies the ending of a log file
Alternatively, if all your full scale images have the same voxel size, it can be specified with 'full_res_voxel_size: float'

voxel_size_log_entry

This specifies the line that gives the voxel size of the original scan
For our Skyscan 1172 it is 'Pixel Size (um)='

full_res_voxel_size

Specifiy the voxel size of all the full scale images. If this option is chosen, do not use log_file_endswith and voxel_size_log_entry


Run LAMA

This will produce a yaml config file in the output directory called pad_info.yaml
LAMA ussually pads volumes before registration to ensure all are the same size. in pad_info.yaml the amount of padding is recorded and the paths to each full size folder is recorded
